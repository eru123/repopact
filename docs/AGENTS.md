# Documentation Instructions

This file applies to everything under `docs/`.

- Keep durable documentation under the correct subsection.
- Update `docs/README.md` when adding, moving, renaming, or deleting docs.
- Do not create task-completion summary files.
- Do not create customer-facing changelogs until the project owner explicitly marks the project as operating/released.
- Use `docs/developers/task-logs.md` for implementation decisions and completed task notes.
- Keep docs practical and current. Remove stale instructions instead of adding historical disclaimers.
- Prefer examples that are safe to copy.
- Redact secrets, personal data, production URLs, and credentials.
