# Developer Documentation Instructions

This file applies to `docs/developers/`.

- Keep implementation workflow, internal design notes, verification notes, and task logs here.
- `task-logs.md` is latest-first.
- Every meaningful code, docs, test, structure, or workflow change must add a short entry to `task-logs.md`.
- Do not create separate task summary files.
- Link durable docs from `docs/README.md`.
