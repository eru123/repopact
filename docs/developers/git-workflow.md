# Git Workflow

Use the current branch by default. Do not create a new branch unless the project owner asks.

Before committing:

1. Confirm changed files are intentional.
2. Confirm documentation reflects behavior and structure changes.
3. Confirm `docs/developers/task-logs.md` has a latest-first entry.
4. Run relevant checks.
5. Confirm the checks completed successfully.

Commit messages:

- Imperative.
- Maximum 150 characters in the subject.
- No co-author trailers.
- No AI signatures or generated-by notices.
- GPG signing is not required.

Push after completing the session unless the project owner says not to push.
