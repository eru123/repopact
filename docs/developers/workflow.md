# Developer Workflow

Use this workflow for meaningful changes:

1. Read `AGENTS.md` and any child instruction files.
2. Understand the existing implementation before editing.
3. Update `docs/plan/specs.md` when requirements change.
4. Update `docs/plan/design.md` when implementation approach changes.
5. Update `docs/plan/tasks.md` as a flat checklist.
6. Make the smallest safe code change.
7. Add or update the smallest relevant verification.
8. Update the nearest owning docs page.
9. Add a latest-first entry to `docs/developers/task-logs.md`.
10. Run relevant checks.
11. Commit unless the owner says not to.

Avoid unrelated formatting and broad rewrites.
